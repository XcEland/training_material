# Peer Assessment Rubric

Score each category from 1 to 5.

| Category | 1 - Weak | 3 - Acceptable | 5 - Strong | Score |
|---|---|---|---|---:|
| Technical soundness | Design is unclear or incomplete | Design works but has gaps | Design is coherent, tested, and maintainable |  |
| Security compliance | Controls are missing | Basic controls included | Least privilege, injection prevention, credential safety, and audit trail are clear |  |
| Measurability | Benefits are vague | Some KPIs included | KPIs, baselines, ROI scenarios, and payback are quantified |  |
| Deployment readiness | No release plan | Basic release steps | Environment gates, rollback, monitoring, and support ownership are clear |  |
| Stakeholder alignment | Audience needs unclear | Some stakeholder value shown | Proposal clearly connects technical work to business and regulatory impact |  |
| Presentation quality | Hard to follow | Understandable | Executive-ready, concise, and defensible |  |
